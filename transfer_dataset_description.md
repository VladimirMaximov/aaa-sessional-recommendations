# Описание датасета

Архив содержит обезличенные данные для задачи ранжирования, а также заголовки и изображения айтемов.

## Состав архива

Ожидаемая структура:

```text
train.parquet
val.parquet
test.parquet
images/
  <имя_файла_изображения>
```

Файлы с данными:

- `train.parquet` - обучающая выборка.
- `val.parquet` - валидационная выборка.
- `test.parquet` - тестовая выборка.

Директория `images/` содержит изображения айтемов, на которые ссылаются parquet-файлы.

## Семантика строк

Одна строка соответствует одному кандидату-айтему в одном контексте ранжирования.

Строки с одинаковым значением `x` относятся к одному и тому же контексту ранжирования. Для расчета ranking-метрик и grouped evaluation нужно использовать `x` как идентификатор группы/query.

## Колонки

Основные колонки:

- `x` - идентификатор контекста ранжирования. Используется для группировки кандидатов из одного запроса/сессии.
- `item_id` - идентификатор айтема. Стабилен внутри датасета и может использоваться для join-ов по айтемам.
- `title` - заголовок айтема.
- `image_link` - имя файла изображения, не полный URL. Пример: `7928527345.jpg`.
- `strong_target` - целевая метка для обучения/ранжирования.

Колонка пользователя:

- В датасете может присутствовать одна из колонок `buyer_user_id`, `user_id` или `x_user_id` в зависимости от схемы выгрузки. Значения в этой колонке обезличены и не содержат исходных идентификаторов пользователей.

Фичи:

- `float_*` - числовые фичи модели.
- `cat_*` - категориальные фичи модели.

Названия фичей намеренно обобщены. Архив не содержит исходные названия фичей или mapping-и.

## Как связать строки с изображениями

Значение `image_link` нужно использовать как имя файла внутри директории `images/`.

Пример:

```python
from pathlib import Path

dataset_root = Path("./dataset")
image_path = dataset_root / "images" / row["image_link"]
```

Если в строке:

```text
image_link = "7928527345.jpg"
```

то соответствующее изображение находится по пути:

```text
images/7928527345.jpg
```

В некоторых строках `image_link` может быть пустым или отсутствующим. Для таких строк соответствующего файла изображения нет.

## Минимальный пример чтения

```python
from pathlib import Path

import pyarrow.parquet as pq

dataset_root = Path("./dataset")
train = pq.read_table(dataset_root / "train.parquet").to_pandas()

row = train.iloc[0]
image_path = dataset_root / "images" / row["image_link"] if row["image_link"] else None

print(row["x"], row["item_id"], row["title"], image_path)
```

## Примечания

- Не нужно объединять `train`, `val` и `test` при оценке качества.
- Для ranking-метрик используйте `x` как ключ группировки.
- Для item-level join-ов внутри датасета используйте `item_id`.
- Для поиска изображения используйте `image_link`.
